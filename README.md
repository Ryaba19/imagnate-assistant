# iMagnate Assistant

Демонстрационный прототип нового контент-центра для iMagnate: заказы с сайта, товары, фото, комплектующие, публикации и AI-агенты в одном понятном рабочем маршруте.

Главная идея: если текущий content manager перегружен и в нем трудно ориентироваться, ассистент может стать либо аккуратной надстройкой рядом с ним, либо фундаментом нового упорядоченного интерфейса.

## Страницы

- `login.html` — вход сотрудников по логину и паролю.
- `index.html` — главная концепции: content manager 2.0, заказы, товары, комплектующие и AI.
- `orders.html` — заказы с сайта: клиент, товар, статус, следующее действие и задача продавцу.
- `tasks.html` — отдельный журнал задач: открытые, закрытые, высокий приоритет.
- `shift.html` — закрытие смены для двух сотрудников и расчет расхождения.
- `inventory.html` — товары: остатки, фото, статус и подготовка карточки для Авито.
- `components.html` — комплектующие: стекла, дисплеи, АКБ, кабели, совместимость и закупка.
- `avito.html` — рабочая страница Авито: профиль магазина, статус подключения и очередь карточек.
- `questions.html` — вопросы сотрудников, на которые отвечает AI-продавец.
- `employees.html` — управление сотрудниками для Лёни и админа.
- `site-import.html` — план внедрения нового content manager и безопасного подключения данных.
- `content.html` — техническая заготовка для будущей публикации на сайт/Telegram.

## Общие файлы

- `assets/css/styles.css` — общий дизайн.
- `assets/js/app.js` — вся логика интерфейса и плавающего AI-ассистента.
- `assets/js/demo-data.js` — тестовые данные для статического запуска.
- `server.js` — локальный сервер и безопасный мост к OpenAI API.
- `.env.example` — пример переменных для реального AI-режима.
- `data/demo-data.json` — основная демо-база в JSON.
- `data/products-template.json` — шаблон реальной выгрузки товаров.
- `data/products-template.csv` — CSV-шаблон реальной выгрузки товаров.
- `database/schema.sql` — схема будущей базы.
- `database/store_assistant_demo.sqlite` — готовая тестовая SQLite-база.

- `docs/feature-audit.md` — ревизия функций: что оставить ядром, что сократить и что нужно для внедрения.

## Как открыть

Открой `login.html` в браузере, войди под нужным сотрудником и дальше пользуйся боковым меню.

## Как запустить с реальной нейронкой

Статический режим через `login.html` продолжает работать без сервера. В нем ассистент отвечает демо-логикой.

Для реального AI-режима нужен Node.js 18+ и API-ключ OpenAI или Claude.

OpenAI:

Windows PowerShell:

```powershell
$env:AI_PROVIDER="openai"
$env:OPENAI_API_KEY="sk-your-key-here"
npm.cmd start
```

macOS Terminal:

```bash
export AI_PROVIDER="openai"
export OPENAI_API_KEY="sk-your-key-here"
npm start
```

Claude:

```powershell
$env:AI_PROVIDER="claude"
$env:ANTHROPIC_API_KEY="sk-ant-your-key-here"
npm.cmd start
```

После запуска открой:

```text
http://127.0.0.1:8787/login.html
```

Если ключ не задан или API недоступен, интерфейс автоматически вернется в демо-режим, чтобы показ не ломался. API-ключ хранится только на сервере и не попадает в браузер.

Дополнительно можно выбрать модель:

```powershell
$env:OPENAI_MODEL="gpt-5.6"
$env:CLAUDE_MODEL="claude-3-5-sonnet-latest"
```

## Контент-центр

Новая логика демо:

```text
Сайт iMagnate
  ↓
Заказ / товар / фото / комплектующие
  ↓
iMagnate Assistant
  ↓
AI-агенты: описание, Авито, задачи, ответы, контроль остатков
  ↓
Сотрудник подтверждает действие
```

Это позволяет обсуждать с Лёней не вторую CRM, а понятный интерфейс вместо хаотичного content manager.

Первый полезный сценарий:

1. С сайта приходит заказ.
2. Ассистент показывает клиента, товар, статус и следующее действие.
3. Продавец создает задачу.
4. Если нужен товар, он добавляется из справочника iMagnate.
5. AI готовит описание, комплект, фото и карточку для сайта/Авито.

Второй сценарий:

1. Комплектующие живут отдельно от товаров.
2. У каждой позиции есть совместимость и минимальный остаток.
3. Ассистент подсвечивает, что надо заказать.
4. Одной кнопкой создается задача на закупку.

Раздел `questions.html` нужен, чтобы сотрудники не дергали Лёню типовыми вопросами. Вопрос сохраняется в журнал, AI-продавец готовит ответ и закрывает карточку. В реальной версии ответы Лёни можно превратить в базу знаний для нейросети.

## Авито

Во вкладке `inventory.html` у каждого товара есть кнопка “Загрузить” в колонке “Авито”.

Во вкладке `avito.html` все сотрудники могут открыть публичный профиль магазина:

`https://www.avito.ru/user/45b3050882d7589ba21bf140dde5c6f8/profile?src=sharing`

Сейчас это демо-черновик:

- подтягивается название товара, цена, категория, город и контакт;
- можно прикрепить фото товара;
- можно указать ссылки на фото для XML-выгрузки;
- можно указать источник фото: живые фото, сайт магазина, поставщик или смешанный набор;
- ассистент показывает предпроверку модерации: мало фото, фото с сайта, каталожные изображения, короткое описание, контакты в тексте и другие риски;
- можно заполнить состояние, комплектацию и описание;
- можно скопировать подготовленный текст;
- можно скачать XML-файл для будущей автозагрузки;
- можно скачать JSON-заявку для будущего API;
- можно смотреть список товаров, которые нужно подготовить для Авито;
- можно обновить демо-статус подключения профиля;
- кнопка “Загрузить на Авито” сохраняет демо-статус, но не отправляет объявление реально.

Для настоящей загрузки понадобится подключение к Авито API, XML-автозагрузке или другому официальному способу интеграции.

Важно: для реальной XML-автозагрузки фотографии должны быть доступны по публичным ссылкам. Локальные фото в демо показываются в предпросмотре, но перед настоящей отправкой их нужно загрузить на сайт, CDN или другое хранилище.

Фото с сайта магазина можно использовать как источник данных, но для Авито лучше готовить живые уникальные фото конкретной единицы товара. Ассистент специально подсвечивает такие карточки как рискованные, чтобы сотрудник успел заменить или дополнить фотографии до публикации.

## Товары и остатки

Раздел `inventory.html` специально упрощен для демо: сотрудник видит товар, статус, остаток, наличие фото, кнопку подготовки к Авито и комментарий. Финансовые колонки и выбор товара из справочника убраны из основного сценария, чтобы не перегружать экран.

Кнопка “Обновить остатки” показывает место будущей синхронизации с CRM, сайтом или регулярной выгрузкой CSV/XLSX.

## Как пересобрать базу

```bash
python scripts/build_demo_db.py
```

Скрипт читает `data/demo-data.json`, применяет `database/schema.sql` и создает `database/store_assistant_demo.sqlite`.

## Подключение сайта iMagnate

Публичный сайт можно использовать для структуры: разделы, категории, акции, статьи, SEO-описания страниц и примерные ссылки на карточки товаров.

Страница `site-import.html` оставлена в проекте как техническая заготовка, но убрана из основного меню демо. Для показа Лёне достаточно объяснить: реальные товары лучше подключать через CRM/API или выгрузку CSV/XLSX, а публичный сайт использовать только как дополнительный источник структуры и контента.

Минимальные поля товара:

- `sku` — артикул или внешний ID;
- `name` — название;
- `category` — категория;
- `price` — цена продажи;
- `stock` — остаток;
- `description` — описание;
- `kit` — комплектация;
- `photoSource` — источник фото: `own`, `site`, `supplier` или `mixed`;
- `photoUrls` — ссылки на фотографии для будущей XML-выгрузки.

Страница `content.html` тоже оставлена как техническая заготовка для будущих публикаций на сайт и Telegram. В текущем демо основной путь публикации вынесен в `inventory.html` и `avito.html`.

Для реальных товаров, цен, остатков, заказов и статусов лучше использовать один из вариантов:

- доступ к CRM/API;
- регулярная выгрузка CSV/XLSX;
- согласованный серверный импорт публичных страниц с ограниченной частотой.

Самый безопасный первый путь — попросить тестовую выгрузку товаров и остатков, а уже потом делать автоматический импорт.

## AI

В проекте есть два режима:

- **Демо-режим** — работает без интернета, сервера и API-ключа.
- **Реальный AI-режим** — `server.js` отправляет запрос в OpenAI Responses API или Claude Messages API и возвращает готовый ответ в интерфейс.

Плавающая кнопка `?` работает как AI-продавец iMagnate: помогает клиенту выбрать товар, подсказывает сотруднику порядок действий и умеет отвечать по текущему контексту магазина. Ассистент не должен выдумывать наличие, скидки, гарантию или внутренние финансовые данные.

Для доведения до внедрения в магазине нужны следующие блоки:

- реальная база товаров и остатков из CRM, сайта или CSV/XLSX;
- хранение пользователей и паролей на сервере;
- журнал действий сотрудников;
- настоящий импорт/экспорт Авито через официальный канал;
- загрузка фото в публичное хранилище для сайта и Авито;
- база знаний для AI: правила магазина, гарантия, trade-in, доставка, ремонт, типовые ответы;
- разграничение AI-ответов: клиенту не показывать закупку, маржу и внутренние комментарии.
