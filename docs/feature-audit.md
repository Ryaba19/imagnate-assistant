# Ревизия функций iMagnate Assistant

## Ключевые функции

1. **AI-продавец**
   - Плавающая кнопка `?`.
   - Работает в демо-режиме без ключа.
   - Поддерживает серверный режим через OpenAI или Claude.
   - Должен стать главным интерфейсом для продавцов и владельца.

2. **Товары и остатки**
   - Остатки по всем позициям.
   - Добавление товара из справочника.
   - Автозаполнение категории, цены, описания и комментария.
   - Это основа будущей реальной базы магазина.

3. **Авито**
   - Подготовка объявления из карточки товара.
   - Фото, описание, комплектация, состояние, цена.
   - Предпроверка рисков модерации.
   - Следующий шаг: официальный канал выгрузки или XML.

4. **Вопросы сотрудников**
   - Сотрудники задают вопросы в системе.
   - Лёня/админ отвечают в карточке.
   - AI-черновик готовит быстрый ответ.
   - Это будущая база знаний для нейросети.

5. **Роли и сотрудники**
   - Лёня и админ видят больше.
   - Продавцы не видят управление сотрудниками и сайт.
   - В боевой версии нужно вынести пароли на сервер.

6. **Закрытие смены**
   - Расчет кассы и расхождений.
   - Полезно, если сейчас часть закрытия идет на бумаге.

## Что можно сократить или спрятать

1. **Страница “Публикации”**
   - Оставить как очередь каналов: сайт, Telegram, Авито.
   - Быстрый генератор текста можно сделать вторичным блоком, потому что основная подготовка идет из товара.

2. **Страница “Сайт”**
   - Не показывать продавцам.
   - Оставить для Лёни/админа как технический раздел импорта.

3. **Главная**
   - Не перегружать карточками.
   - Оставить 3-5 главных сигналов: смена, остатки, проблемы, вопросы, Авито.

4. **Демо-доступы**
   - На этапе показа удобно.
   - В рабочей версии таблицу логинов и паролей нужно убрать.

5. **Финансовые показатели**
   - Оставить только Лёне и админу.
   - Продавцам показывать задачи, товары, вопросы и публикации.

## Что нужно для внедрения в магазине

1. Реальная база товаров и остатков: CRM, API, CSV/XLSX или ручной импорт.
2. Серверная авторизация: пароли не должны храниться в браузере.
3. Хранилище фотографий: сайт, CDN или облако для публичных ссылок Авито.
4. Журнал действий: кто добавил товар, изменил остаток, создал объявление.
5. База знаний для AI: правила гарантии, trade-in, ремонт, доставка, типовые ответы.
6. Интеграция Авито: официальный API/XML/автозагрузка.
7. Настройки магазина: город, контакты, график, шаблоны описаний.
8. Резервное копирование базы.

## Текущий вывод

Демо уже можно показывать как концепт рабочего ассистента. Для настоящего запуска первой полезной версией лучше выбрать узкий MVP:

**Товары + Авито + Вопросы сотрудников + AI-продавец.**

Остальное можно оставить как вторую волну.
