# Обновление с базой данных PostgreSQL

Теперь заявки с сайта хранятся в настоящей базе данных и переживают любые перезапуски сервера. Если базы нет — сервер автоматически работает по-старому (файл leads.json).

## Шаг 1. Создать базу на Render (один раз, ~2 минуты)

1. [dashboard.render.com](https://dashboard.render.com) → кнопка **New +** → **PostgreSQL**.
2. Name — например `imagnate-db`, регион тот же, что у сервиса, план **Free** → Create Database.
3. Когда база создастся, на её странице найдите **Internal Database URL** (строка вида `postgresql://…`) и скопируйте её.

Если база у вас уже есть — просто возьмите её Internal Database URL.

## Шаг 2. Подключить базу к сервису

1. Откройте сервис `imagnate-assistant` → **Environment**.
2. Добавьте переменную: ключ `DATABASE_URL`, значение — скопированный Internal Database URL.
3. `STORE_TOKEN` оставьте как есть.

## Шаг 3. Обновить файлы в репозитории

Загрузите в ветку `render-store-control-real` (Add file → Upload files, файлы по отдельности):

- `server.js` и `storecontrolapi.js` — новый сервер с поддержкой БД (два одинаковых файла под разными именами — чтобы запускался при любой настройке)
- `package.json` — теперь с зависимостью `pg`
- `index.html` — свежая ERP (с автосохранением в браузере)
- `form.html` — если ещё не заливали

## Шаг 4. Настройки сборки

Settings → Build & Deploy:

- **Build Command:** `npm install`  ← важно! иначе модуль базы не установится
- **Start Command:** `node server.js` (или node storecontrolapi.js — теперь без разницы)

Сохраните — Render передеплоит.

## Проверка

Откройте `https://imagnate-assistant.onrender.com/api/health`:

- `{"ok":true,"db":"postgres"}` — база подключена, заявки в PostgreSQL ✅
- `{"ok":true,"db":"file"}` — сервер работает, но базу не увидел: проверьте DATABASE_URL и Build Command

Таблица `site_leads` создаётся автоматически при первом запуске. ERP забирает заявки как раньше — ей неважно, откуда сервер их отдаёт.

## Что где хранится теперь

| Данные | Где | Переживает |
|---|---|---|
| Заявки с сайта | PostgreSQL на Render | перезапуски, передеплои — всё |
| Данные ERP (заказы, задачи, поставки…) | Браузер (автосохранение) | перезагрузку страницы на этом устройстве |

Следующий возможный шаг — хранить данные ERP тоже в базе, чтобы они были общими для всех устройств и сотрудников. Скажите, когда будете готовы.
