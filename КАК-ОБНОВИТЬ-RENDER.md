# Обновление imagnate-assistant.onrender.com — конечный вариант

В этой папке всё, что должно лежать на Render. После обновления по одному адресу будет жить вся связка:

| Адрес | Что там |
|---|---|
| `https://imagnate-assistant.onrender.com/` | Сама ERP (свежая версия) |
| `https://imagnate-assistant.onrender.com/form` | Страница с формой заявки — её адрес можно давать клиентам или вставить ссылкой на сайт |
| `https://imagnate-assistant.onrender.com/api` | API: приём заявок и выдача их в ERP |

## Файлы

- `server.js` — сервер (Node.js, без зависимостей: `npm install` не нужен)
- `index.html` — ERP-система (конечная версия)
- `form.html` — страница формы заявки (уже настроена на этот же сервер)
- `package.json` — чтобы Render знал команду запуска

## Как обновить сервис

1. Замените этими файлами содержимое репозитория/папки, из которой развёрнут сервис на Render, и сделайте деплой (push в репозиторий или Manual Deploy → Deploy latest commit).
2. Настройки сервиса: **Build Command** — пусто, **Start Command** — `node server.js`.
3. В **Environment** должна быть переменная `STORE_TOKEN` — ваш секретный токен (он у вас уже задан; если нет — добавьте длинную случайную строку).

## Как подключить канал в ERP

Откройте `https://imagnate-assistant.onrender.com/` → CRM → Интеграции → **Сайт** → Подключить. Адрес API уже подставлен автоматически (`/api`), останется ввести только токен — тот же, что в `STORE_TOKEN` на Render. Дальше заявки с `/form` сами появляются в воронке CRM (проверка раз в минуту, есть кнопка «Синхронизировать»).

Если открываете ERP не с Render, а из локального файла — в поле адреса подставится полный `https://imagnate-assistant.onrender.com/api`, работает одинаково.

## Два нюанса бесплатного Render

- Сервис «засыпает» после 15 минут без посещений: первый запрос после сна отвечает до минуты. Заявка с формы при этом не теряется — форма честно ждёт ответа.
- Диск не сохраняется между перезапусками: файл `leads.json` может обнулиться. Это не критично — ERP забирает заявки каждую минуту, так что всё полученное уже в системе. Для 100% сохранности можно подключить Render Disk (платно) и задать переменную `DATA_DIR` с путём к нему.
